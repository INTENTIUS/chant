The Temporal lexicon provides **4** rules: 2 lint rules and 2 post-synth checks.

## Lint Rules

| ID | Severity | Category | Description |

Post-synth checks validate the serialized output after the build pipeline completes.

| ID | Description |
|----|-------------|
| `TMP010` | TemporalSchedule cron expressions must be valid 5- or 6-field cron syntax |
| `TMP011` | SearchAttribute.namespace must reference a declared TemporalNamespace entity |
