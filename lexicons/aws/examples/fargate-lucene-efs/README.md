# Fargate + Lucene/Solr + EFS

Three CloudFormation stacks wire together a VPC, an EFS-backed Solr service on Fargate, and Lambda connectors that index data from DynamoDB and S3. A fourth stack ‚Äî a local Docker Compose file generated by chant ‚Äî mirrors the same data flow so you can iterate on queries and indexing without touching AWS.

## Agent walkthrough

Paste this to Claude Code in the repo root:

```
Deploy the fargate-lucene-efs example to AWS.
My AWS region is us-east-1.
```

The agent builds all stacks, smoke-tests locally with Docker, then deploys the three AWS stacks in order. The phase-by-phase breakdown below shows what it does under the hood, for reference.

### Phase 0 ‚Äî Local smoke test

```bash
just up && just -f docker/justfile verify
```

Starts DynamoDB Local + MinIO + Solr via Docker Compose, then runs the E2E relay tests: insert a DynamoDB row ‚Üí confirm it appears in Solr, upload a MinIO file ‚Üí confirm it's indexed. All 3 pass in ~20 seconds.

### Phase 1 ‚Äî Build

```bash
just build-all
```

Runs `chant build` in each of the four stacks, producing three CloudFormation templates (`infra/template.json`, `solr/template.json`, `connectors/template.json`) and a Docker Compose file (`docker/dist/docker-compose.yml`).

### Phase 2 ‚Äî Deploy infra

```bash
just -f infra/justfile deploy
```

Creates a VPC with public and private subnets, NAT gateway, ALB, ECS cluster, EFS filesystem, and EFS access point. Key outputs passed to the next stack: `clusterArn`, `listenerArn`, `efsId`, `accessPointId`, `albDnsName`, and more.

### Phase 3 ‚Äî Deploy Solr

```bash
just -f solr/justfile deploy
```

Reads the infra stack outputs and passes them as CloudFormation parameter overrides. Deploys a Fargate task running `solr:9` with `/var/solr` mounted on the EFS access point (POSIX UID/GID 8983). Solr is reachable at `http://<albDnsName>/solr`.

### Phase 4 ‚Äî Deploy connectors

```bash
just -f connectors/justfile deploy
```

Reads the solr stack outputs. Creates a DynamoDB table with Streams enabled and a Lambda function (inline Python) that relays inserts/deletes to Solr. Also creates an S3 bucket and a Lambda that indexes uploaded objects into Solr.

## Prerequisites

- AWS CLI configured (`aws sts get-caller-identity` should return your account)
- Docker Desktop running (for Phase 0)
- Bun installed ‚Äî [https://bun.sh](https://bun.sh)
- `npm install` run from the repo root

## Cost

NAT gateway (~$32/mo) + ALB (~$16/mo) + EFS (~$0.30/GB-mo) ‚âà $48+/mo while the stacks are up. Tear down after testing.

## Tear down

```
Tear down the fargate-lucene-efs stacks in us-east-1.
```

Or manually, in reverse order:

```bash
just teardown-all
```
