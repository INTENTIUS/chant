The Slurm lexicon brings infrastructure-as-code discipline to HPC cluster management.
Define your `slurm.conf` in TypeScript, get static analysis, and generate deterministic
configuration files ready to distribute to all nodes.

## Why

`slurm.conf` must be **identical on every cluster node** ‚Äî but there is no IaC tooling
for it. Sites use Ansible/Jinja2 templates, but there is no plan/apply workflow, no
drift detection, and no type-safe schema. A typo in `SelectTypeParameters` silently
disables fair memory allocation across hundreds of nodes.

Chant fills this gap: TypeScript source is the single source of truth, CI builds
deterministic output, and 21 post-synth checks catch production gotchas before deploy.

| **AI skills** | Generate jobs, clusters, explain fairshare |

## Quick start

```bash
chant init --lexicon slurm my-cluster
cd my-cluster
chant build src --lexicon slurm
```

See [Getting Started](/getting-started) for a full walkthrough.

## Key catches

The post-synth checks encode real production failures:

- **SLR013** ‚Äî `AuthType=auth/none` exposes slurmctld to unauthenticated job submission
- **SLR017** ‚Äî `DefMemPerCPU` + `DefMemPerNode` on the same partition is a silent config error
- **SLR018** ‚Äî Missing `GresTypes=gpu` means GPUs are never allocated to jobs
- **SLR021** ‚Äî `PriorityWeightFairshare=10` makes fairshare irrelevant vs age/job-size weights
- **SLR024** ‚Äî `SuspendProgram` without `ResumeProgram` permanently suspends nodes
- **SLR027** ‚Äî `proctrack/cgroup` without `cgroup.conf` means `ConstrainRAMSpace=false` ‚Äî runaway jobs OOM the node
- **SLR029** ‚Äî `AcctGatherEnergyType=rapl` with `AcctGatherNodeFreq‚â•300` produces silent counter overflow (data corruption)
