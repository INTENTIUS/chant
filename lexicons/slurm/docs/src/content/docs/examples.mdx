## gpu-cluster

A GPU-focused cluster using the `GpuPartition` composite. Demonstrates:

- `GresTypes: "gpu"` on the Cluster
- `GpuPartition` composite for `gpu[001-008]` with A100Г—8 per node
- Multifactor priority with `PriorityWeightFairshare: 10000`
- Login partition separate from the GPU partition

## multi-partition

A multi-partition cluster with accounting and fairshare. Demonstrates:

- `AccountingStorageType: "accounting_storage/slurmdbd"` wired to a local slurmdbd
- `AccountingStorageEnforce: "associations,limits,qos"` вЂ” enforces QOS limits
- Two node types: standard CPU and high-memory
- Two partitions: `cpu` (default) and `himem`


## AWS HPC cluster

A production-grade EDA HPC cluster on AWS with:

- FSx for Lustre (`PERSISTENT_2`) shared storage
- Aurora MySQL Serverless v2 for slurmdbd
- EFA networking with cluster placement groups
- Spot fleet with ASG lifecycle hooks + EventBridge Lambda for spot interruption handling
- SSM Automation document for orchestrated deploy
- EDA license management (synthesis, simulation, DRC)
