The Slurm lexicon uses a routing serializer that dispatches on entity type prefix.

## Conf resources → slurm.conf

Resources with type prefix `Slurm::Conf::*` serialize into the primary `slurm.conf` output,
with the following exceptions that route to separate files:

| Resource | Output file | Format |
|---|---|---|
| `Cluster` | `slurm.conf` (primary) | Top-level key=value lines |
| `Node` | `slurm.conf` (primary) | `NodeName=<pattern> CPUs=... RealMemory=...` stanza |
| `Partition` | `slurm.conf` (primary) | `PartitionName=<name> Nodes=... MaxTime=...` stanza |
| `License` | `slurm.conf` (primary) | Aggregated into a single `Licenses=name1:count1,...` line |
| `GresNode` | `gres.conf` | `NodeName=... Name=... AutoDetect=...` stanza |
| `CgroupConf` | `cgroup.conf` | Flat `Key=value` file (booleans as `true`/`false`) |
| `Switch` | `topology.conf` | `SwitchName=... Nodes=...` stanza |

Example output:

```conf
# slurm.conf (generated by chant)
ClusterName=hpc
ControlMachine=head01
AuthType=auth/munge
SelectType=select/cons_tres
SelectTypeParameters=CR_Core_Memory
GresTypes=gpu
Licenses=eda_synth:50,eda_sim:200

NodeName=cpu[001-016] CPUs=96 RealMemory=196608 State=UNKNOWN
NodeName=gpu[001-004] CPUs=96 RealMemory=1048576 Gres=gpu:a100:8 State=UNKNOWN

PartitionName=cpu Nodes=cpu[001-016] Default=YES MaxTime=7-00:00:00
PartitionName=gpu Nodes=gpu[001-004] Default=NO MaxTime=1-00:00:00
```

**Order:** Cluster global settings → NodeName stanzas → PartitionName stanzas.
Multiple License entities are merged into a single `Licenses=` line.

## REST resources → JSON files

Resources with type prefix `Slurm::Rest::*` serialize to JSON files alongside `slurm.conf`:

| Resource | File path |
|---|---|
| `Job` | `jobs/<name>.json` |
| `Reservation` | `reservations/<name>.json` |
| `QoS` | `qos/<name>.json` |

When REST resources are present, the build returns a `SerializerResult` with:
- `primary` — the `slurm.conf` text
- `files` — a map of path → JSON content

```
dist/
├── slurm.conf          (primary)
├── gres.conf           (GresNode entities)
├── cgroup.conf         (CgroupConf entity)
├── topology.conf       (Switch entities)
├── jobs/
│   └── llm-train.json
└── qos/
    └── high-priority.json
```

When any of `GresNode`, `CgroupConf`, or `Switch` entities are present, the build returns
a `SerializerResult` with `{ primary, files }` instead of a plain string.

## rulePrefix

The serializer uses `rulePrefix: "SLR"`. Lint rules and post-synth checks are
namespaced under this prefix (SLRC001, SLRS001, SLR010–SLR030).
