# Fargate + Lucene/Solr + EFS

Three CloudFormation stacks wire together a VPC, an EFS-backed Solr service on Fargate, and Lambda connectors that index data from DynamoDB and S3. A fourth stack — a local Docker Compose file generated by chant — mirrors the same data flow so you can iterate on queries and indexing without touching AWS.

## Agent walkthrough

The lexicon packages ship skills for agent-guided deployment. After `npm install`, your agent has access to:

| Skill | Package | Purpose |
|-------|---------|---------|
| `chant-aws` | `@intentius/chant-lexicon-aws` | CloudFormation lifecycle: build, validate, change sets, deploy, rollback |
| `chant-docker` | `@intentius/chant-lexicon-docker` | Docker Compose lifecycle: build, up, down, validate |
| `chant-docker-patterns` | `@intentius/chant-lexicon-docker` | Common Docker Compose patterns: databases, volumes, healthchecks |

Paste this to Claude Code from the repo root:

```
Deploy the fargate-lucene-efs example.
The example is in examples/fargate-lucene-efs.
My AWS region is us-east-1.

Run npm install from the repo root first, then follow the phase-by-phase
instructions in examples/fargate-lucene-efs/README.md.
Set AWS_REGION=us-east-1 before running any just commands.
```

The agent uses `chant-aws` for CloudFormation and `chant-docker` for the local smoke test, then deploys the three AWS stacks in order. The phase-by-phase breakdown below shows what it does under the hood, for reference.

### Phase 0 — Install and local smoke test

```bash
# From repo root
npm install
cd examples/fargate-lucene-efs
export AWS_REGION=us-east-1
just up && just -f docker/justfile verify
```

Starts DynamoDB Local + MinIO + Solr via Docker Compose, then runs the E2E relay tests: insert a DynamoDB row → confirm it appears in Solr, upload a MinIO file → confirm it's indexed. All 3 pass in ~20 seconds.

### Phase 1 — Build

```bash
just build-all
```

Runs `chant build` in each of the four stacks, producing three CloudFormation templates (`infra/template.json`, `solr/template.json`, `connectors/template.json`) and a Docker Compose file (`docker/dist/docker-compose.yml`).

### Phase 2 — Deploy infra

```bash
just -f infra/justfile deploy
```

Creates a VPC with public and private subnets, NAT gateway, ALB, ECS cluster, EFS filesystem, and EFS access point. Key outputs passed to the next stack: `clusterArn`, `listenerArn`, `efsId`, `accessPointId`, `albDnsName`, and more.

### Phase 3 — Deploy Solr

```bash
just -f solr/justfile deploy
```

Reads the infra stack outputs and passes them as CloudFormation parameter overrides. Deploys a Fargate task running `solr:9` with `/var/solr` mounted on the EFS access point (POSIX UID/GID 8983). Solr is reachable at `http://<albDnsName>/solr`.

### Phase 4 — Deploy connectors

```bash
just -f connectors/justfile deploy
```

Reads the solr stack outputs. Creates a DynamoDB table with Streams enabled and a Lambda function (inline Python) that relays inserts/deletes to Solr. Also creates an S3 bucket and a Lambda that indexes uploaded objects into Solr.

## Prerequisites

- AWS CLI configured (`aws sts get-caller-identity` should return your account)
- Docker Desktop running (for Phase 0)
- Node.js 20+ — [https://nodejs.org](https://nodejs.org)
- `npm install` run from the repo root

## Cost

NAT gateway (~$32/mo) + ALB (~$16/mo) + EFS (~$0.30/GB-mo) ≈ $48+/mo while the stacks are up. Tear down after testing.

## Tear down

```
Tear down the fargate-lucene-efs stacks.
The example is in examples/fargate-lucene-efs. AWS_REGION=us-east-1.
```

Or manually, in reverse order:

```bash
just teardown-all
```
