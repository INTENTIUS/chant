chant provides a CLI for building, linting, and managing infrastructure definitions.

## Installation

The CLI is included in `@intentius/chant`. Install a lexicon package to get everything:

```bash
npm install --save-dev @intentius/chant-lexicon-aws
```

## Usage

```
chant <command> [options] [path]
```

## Commands

| Command | Description |
|---------|-------------|
| [`init`](/chant/cli/init/) | Scaffold a new chant project |
| [`init lexicon`](/chant/cli/init-lexicon/) | Scaffold a new lexicon plugin project |
| [`build`](/chant/cli/build/) | Synthesize and write infrastructure output |
| [`lint`](/chant/cli/lint/) | Lint intent definitions with semantic rules |
| [`list`](/chant/cli/list/) | List exported declarations in a project |
| [`import`](/chant/cli/import/) | Import infrastructure templates and generate TypeScript |
| [`update`](/chant/cli/update/) | Sync lexicon type definitions |
| [`doctor`](/chant/cli/doctor/) | Check project health and configuration |

## Lexicon Development Commands

These commands are for **lexicon authors** — project users don't typically need them.

| Command | Description |
|---------|-------------|
| [`dev generate`](/chant/cli/generate/) | Generate lexicon artifacts (+ validate + coverage) |
| [`dev publish`](/chant/cli/package/) | Package lexicon for distribution |
| [`dev onboard`](/chant/cli/onboard/) | Onboard a new lexicon plugin |
| [`dev check-lexicon`](/chant/cli/check-lexicon/) | Validate a lexicon bundle |

## Op Commands

Run named Temporal workflows defined in `*.op.ts` files. Requires `@intentius/chant-lexicon-temporal` and a running Temporal server.

| Command | Description |
|---------|-------------|
| [`run <name>`](/chant/cli/run/) | Start an Op workflow (spawns worker + submits to Temporal) |
| [`run list`](/chant/cli/run/) | List all Ops with current run status |
| [`run status <name>`](/chant/cli/run/) | Show current workflow run state |
| [`run signal <name> <signal>`](/chant/cli/run/) | Send a named signal to unblock a gate step |
| [`run cancel <name>`](/chant/cli/run/) | Cancel the active workflow run |
| [`run log <name>`](/chant/cli/run/) | Show run history for an Op |
| [`graph`](/chant/cli/graph/) | Print the Op dependency graph |

## State Commands

Capture and compare deployed infrastructure state using orphan git branches.

| Command | Description |
|---------|-------------|
| [`state snapshot <env>`](/chant/cli/state/) | Query API, save metadata to orphan branch |
| [`state show <env>`](/chant/cli/state/) | Show latest state snapshot |
| [`state diff <env>`](/chant/cli/state/) | Compare current build against last snapshot |
| [`state log [env]`](/chant/cli/state/) | History of state snapshots |

## Server Commands

| Command | Description |
|---------|-------------|
| [`serve lsp`](/chant/cli/lsp/) | Start the LSP server for editor integration |
| [`serve mcp`](/chant/cli/mcp/) | Start MCP server for AI agent integration |

## Common Workflows

```bash
# Initialize and build a project
chant init my-infra --lexicon aws
cd my-infra
chant build

# Lint definitions
chant lint

# Import existing templates
chant import template.json

# Check project health
chant doctor
```

## File Discovery

The CLI automatically discovers infrastructure files based on these rules:

**Included:**
- All `.ts` files in the specified directory and subdirectories

**Excluded:**
- Files in `node_modules/` directories
- Test files: `*.test.ts`, `*.spec.ts`
- TypeScript declaration files: `*.d.ts`

## Error Handling

The CLI provides structured error output for different types of errors:

### Discovery Errors

Reported when files can't be loaded or imported. Common causes: missing dependencies, syntax errors, invalid import paths, circular dependencies.

### Build Errors

Reported during serialization. Common causes: circular dependencies between resources, invalid attribute references, missing required properties.

## Global Options

### `--verbose, -v`

Show stack traces on errors.

### `--help, -h`

Display help information for the CLI.

```bash
chant --help
```
