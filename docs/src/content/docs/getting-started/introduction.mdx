---

chant is a TypeScript-first infrastructure tool. You define cloud resources as plain TypeScript objects, chant type-checks them against a provider lexicon, and emits deployment-ready output. No YAML, no string templating, no runtime execution.

```
TypeScript source → chant build → provider-specific output
```

You get real `import`/`export`, editor autocompletion, and semantic lint rules that catch misconfigurations before deployment — all in a single local pass.

## Choose Your Path

- **Synthesis only** — You have CI, ArgoCD, Terraform, or another deployment tool. chant produces typed, linted artifacts and hands off. [Quick Start](/chant/getting-started/quick-start/) covers everything. Lexicons are available for [AWS](/lexicons/aws/), [Azure](/lexicons/azure/), [GCP](/lexicons/gcp/), [Kubernetes](/lexicons/k8s/), [Helm](/lexicons/helm/), [Docker](/lexicons/docker/), [GitHub Actions](/lexicons/github/), [GitLab CI](/lexicons/gitlab/), [Flyway](/lexicons/flyway/), [Temporal](/lexicons/temporal/), and [Slurm](/lexicons/slurm/).
- **Synthesis + deployment** — You want durable deployments with phase visibility, gate steps, and rollback. Declare phases in `*.op.ts`, `chant build` generates Temporal worker code, `chant run` starts the workflow. Start with [Quick Start](/chant/getting-started/quick-start/), then the [Ops guide](/chant/guide/ops/). Not sure if Ops fits? See [Choosing Your Deployment Model](/chant/concepts/deployment-paths/).
- **Create a lexicon** — [Philosophy](/chant/concepts/philosophy/) for the design rationale, then [Lexicon Authoring](/chant/lexicon-authoring/overview/) to build a plugin.
- **Contribute to chant** — [Development Setup](/chant/contributing/development/), then the [Architecture Overview](/chant/architecture/overview/).
